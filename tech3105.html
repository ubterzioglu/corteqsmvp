Pull Workflow

  • 1GitHub API ile son commit'leri tara — sadece yeni/değişmiş dosyaları belirle.
  • 2Lokal versiyonla diff al. OVERLAY değişikliklerini eziyor mu? Auth, App.tsx veya migration etkileniyor mu?
  • 3Migration varsa: tarih kontrolü yap, CREATE TABLE IF NOT EXISTS kullanıldığından emin ol, supabase/migrations/ sonuna ekle.
  • 4Çakışmayan dosyaları kopyala. App.tsx için route ekle (override etme, ekle). Auth değişikliklerini önce audit et.
  • 5Doğrula: npm run build → npm run test → etkilenen sayfayı browser'da kontrol et.
