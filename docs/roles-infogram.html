01

Sistem Mimarisi — Genel Bakış

RolesGo, tüm kullanıcı rolleri, feature'lar, attribute'lar ve onay akışlarını tek bir mimaride yönetir.

✅
Tek sistem, tek kaynak
Tüm kullanıcı verileri user_role_assignments + user_profile_attributes'ta saklanır. Her kullanıcı sisteme bireysel rolüyle girer. Admin panelinden rol, feature ve attribute değişiklikleri yönetilir. Güvenlik Frontend Guard + RLS + RPC üçlüsüyle sağlanır.

Tablo adı: public.user_role_assignments

🔑
Admin tespiti
is_admin(uid) fonksiyonu user_role_assignments'a bakarak r.key ilike 'Admin_%' koşulunu sağlayan bir rol arar. Admin rolü atamak için: admin_set_user_role(user_id, 'Admin_SuperAdmin')
💡
Yeni rol ailesi eklemek
Migration yazmak gerekmez. Sadece roles tablosuna satır eklenir. Yeni rol otomatik olarak bireysel baseline'ının tüm feature flag'lerini ve attribute kurallarını miras alır (backfill migration idempotent çalışır). Admin rolü için key'in Admin_ ile başlaması yeterli.

03

Kayıt & Rol Atama Akışı

Yeni kullanıcı otomatik olarak bireysel rolüyle başlar. Rol değişikliği admin onayı gerektirir.

İlk Kayıt Akışı

1
Google OAuth ile giriş
Kullanıcı Google hesabıyla giriş yapar. auth.users'a kayıt oluşur.
2
Bireysel rol atanır
user_role_assignments'a bireysel rolü otomatik eklenir.
3
/profile sayfasına yönlendirme
Kullanıcı profil bilgilerini tamamlar, isterse rol başvurusu açar.

Rol Başvurusu Akışı

1
Kullanıcı rol seçer
/profile sayfasından hedef rolü seçer, not yazar ve başvuru gönderir.
2
approval_requests'e düşer
role_change tipiyle kayıt açılır. Aynı role ikinci pending başvuru açılamaz.
3
Admin karar verir
Onaylanırsa user_role_assignments otomatik güncellenir. Tüm işlem audit log'a yazılır.
ℹ️
Feature override — rol değiştirmeden yetki vermek
Admin, bir kullanıcıya rolünden bağımsız özel feature açabilir. Örneğin bireysel kullanıcıya events.create = true override verilebilir. Rol değişmez, sadece o feature açılır. Bunun için admin_set_user_feature_override_detailed() kullanılır ve neden alanı doldurulur.

05

Feature Çözümleme Hiyerarşisi

"Bu feature aktif mi?" sorusu get_current_user_features() RPC'sinde 2 adımda cevap bulur.

⚠️
Feature durumunu kendiniz hesaplamayın
useFeatureFlags() hook'unu veya get_current_user_features() RPC'sini kullanın. Kullanıcının user_role_assignments'ta bir kaydı olması şarttır — kayıt yoksa tüm feature'lar kapalı döner.
1
Kişisel Override (En Yüksek Öncelik)
Admin bu kullanıcı için özellikle bir istisna tanımladıysa — bu kazanır. Her şeyi ezer. reason alanında neden override edildiği saklanır.
user_feature_overrides
↓ bulunamazsa
2
Rol Bazlı Ayar
Kullanıcının rolü için bu feature açık/kapalı ayarlandıysa kullanılır. Rol, user_role_assignments'tan alınır.
role_feature_flags
↓ hâlâ bulunamazsa
3
Varsayılan: Kapalı
Hiçbir kayıt bulunamadıysa feature kapalı sayılır. is_active_globally = false tüm zinciri override eder — feature'ı sistem genelinde kapatır.
false (fallback)
🔍
get_current_user_features() mantığı
effective_role CTE: user_role_assignments'tan kullanıcının rolünü al.
Feature listesi: feature_catalog'dan scope_role = '*' veya scope_role = role_key olanları al.
Sonuç: fc.is_active_globally AND coalesce(ufo.is_enabled, rff.is_enabled, false)
Source: 'override' → 'role_default' → 'fallback'
06

Attribute Sistemi

Kullanıcı profilindeki tüm veriler user_profile_attributes'ta dinamik olarak saklanır.

💡
Attribute nedir?
Profil tablosuna kolon eklemek yerine dinamik "özellikler" sistemi kullanılır. Tüm profil alanları attribute_catalog'da tanımlı ve değerleri user_profile_attributes'ta saklanır. Sistem attribute'ları (is_system=true): full_name, country, city, profile_photo_url, bio_short. Rol-özel attribute'lar: interests, expertise_area, business_category, organization_type, main_platform, ambassador_city. Ek alanlar: avatar_url, phone, profession, school, address, cv_path, presentation_path, business_name, business_sector, is_volunteer_mentor ve daha fazlası.

07

Onay (Approval) State Machine

Hassas işlemler doğrudan gerçekleşmiyor — önce admin onayı gerekiyor.

🔄
State Machine nedir?
Bir istek sadece belirli durumlar arasında geçiş yapabilir: PENDING → APPROVED veya PENDING → REJECTED. APPROVED → PENDING olamaz. Bu kurallar database'de zorlanıyor. approval_requests tablosunda aynı (user_id, request_type, target_*) kombinasyonu için yalnızca bir PENDING istek olabilir — unique index bunu garanti eder.

10

RPC Fonksiyonları Referansı

Kritik yazma işlemleri doğrudan tablo update'i yapmaz — RPC üzerinden geçer. Audit log DB/RPC tarafında tutulur.

Fonksiyon Ne Yapar Parametre
get_current_user_profile() Tam profil: rol, feature'lar, attribute'lar, pending istekler, profil completion — (mevcut kullanıcı)
get_current_user_features() Feature key + is_enabled + source listesi (2 katman: override → role_flag) — (mevcut kullanıcı)
update_profile_attribute() Attribute güncelle — approval gerekiyorsa approval_request oluşturur attribute_key, attribute_value, visibility
submit_role_change_request() Rol değiştirme isteği gönder target_role_key, note
submit_feature_request() Feature erişim isteği gönder feature_key, payload
is_admin(uid) Admin kontrolü — user_role_assignments'ta Admin_% key'li rol var mı? uid (UUID)
admin_set_user_role() Kullanıcıya rol ata (user_role_assignments upsert) + audit log user_id, role_key
admin_set_role_feature_flag() Rol için feature aç/kapat + audit log role_key, feature_key, is_enabled
admin_set_user_feature_override_detailed() Kişisel feature override + reason kaydı + audit log user_id, feature_key, is_enabled, reason
admin_set_user_feature_override() Kişisel feature override (reason'sız) user_id, feature_key, is_enabled
admin_clear_user_feature_override() Kişisel feature override'ı siler + audit log user_id, feature_key
admin_review_approval_request() Onay kararı ver + otomatik uygula (rol, attribute veya feature günceller) request_id, 'approved'|'rejected', note
admin_set_attribute_rule() Attribute kuralını güncelle + audit log role_key, attribute_key, rule_payload
admin_set_feature_global_state() Feature'ı tüm sistemde aç/kapat feature_key, is_active_globally
list_public_directory_profiles() Public dizin listesi — feature_state CTE ile feature kontrolü yapar search_text, role_filter, country_filter, city_filter, featured_only, verified_only
🚨
Doğrudan tablo erişimi yasak
supabase.from('user_role_assignments').update() — RLS engeller, admin_set_user_role() kullanın
supabase.from('role_feature_flags').update() — RLS engeller, admin_set_role_feature_flag() kullanın
supabase.from('user_profile_attributes').insert() — update_profile_attribute() RPC kullanın
