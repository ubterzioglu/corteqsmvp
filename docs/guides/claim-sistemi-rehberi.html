🌟 Claim Sistemi Nedir?

Claim sistemi, birinin sahip olduğu bir şeyi (işletme, danışman profili, vb.) sistemde talep etmesine izin verir.

Basit örnek:
ABC Restoran eski bir katalog item'i. Hiç kimse oluşturmadı, sitede yok. Gerçek restaurant sahibi bulundu ve "bu benim işletmem" dedi. Claim sistemi bunu kontrol edip, sahibiliği verir.

❓ Ne Zaman Claim Yapılır?

Şu durumlarda claim yapılır:

Durum Neden? Örnek
Eski/İthal Item Sistemde var ama kimse oluşturmadı Eski restoran listesi, ithal edilen işletmeler
Sahipsiz Profil Profil var ama sahibi giriş yapmadı Gerçek danışman, profili buldu
Doğrulama Sahipliği kanıtlamak gerekli Sertifika, ruhsat, belgeler

👣 Adım Adım: Nasıl Claim Yapılır?

1️⃣ Sahip Katalogu Bulur
"ABC Restoran" var mı diye arar.
2️⃣ "Bu Benim" Butonuna Tıklar
"Claim This Business" veya "Bu Benim" butonuna tıklar.
3️⃣ Talep Formu Doldurur
  • Claim Tipi: "ownership" (sahiplik)
  • Kanıt (Evidence): Fotoğraf, sertifika, ruhsat dosyası (JSON)
  • Not: Neden bu benim? Açıklama
4️⃣ Gönderir
Talep sisteme kaydedilir. Status = pending
5️⃣ Admin İnceliyor
Moderator talebi görerek:
  • Kanıtı kontrol eder
  • Onaylar (approved) veya Reddeder (rejected)
✅ Onaylanırsa
  • Sahiplik talep edene verilir
  • Item'in sahibi değişir
  • Kişi artık o item'i yönetebilir

🔄 Akış Diyagramı

KULLANICI ↓ "Bu benim işletmem" diye claim yapıyor ↓ SISTEM: Talep oluşturuyor (status = pending) ↓ ADMIN TARAFINDA ├─ Kanıt inceleniyor ├─ Sorular sorulabiliyor └─ Karar veriliyor ↓ DECİSİON ├─ approved → Sahiplik verilir ✅ ├─ rejected → Talep reddedilir ❌ └─ cancelled → İptal edilir ⛔ ↓ SONUÇ ├─ Approved: Kullanıcı artık sahibi ├─ Rejected: Item'in sahibi yok └─ Cancelled: Talep iptal

📖 Gerçek Örnek: Restaurant Sahibi

Senaryo

Mehmet'in "Mehmet's Kebab" adında bir restoran var. Sitede bu restaurant zaten var (eski listede). Mehmet giriş yapıyor ve "bu benim" demek istiyor.

Adımlar

Mehmet'in Yaptığı:
  • Restoran katalogunu bulur
  • "Bu İşletmem" butonuna tıklar
  • Form doldurur:
    • Claim Type: "ownership"
    • Evidence: Ruhsat fotoğrafı (.jpg), işletme belgesi
    • Note: "20 yıldır bu restoranda çalışıyorum, ben sahibiyim"
  • Gönderir
Sistem'in Yaptığı:
  • Talep kaydediliyor: catalog_claim_requests tablosuna
  • Status: pending
  • Audit log'a yazılıyor
  • Admin paneline düşüyor
Admin'in Yaptığı (Şehir Moderatörü):
  • /admin/approvals → Pending claims sekmesi
  • Mehmet'in talebini açıyor
  • Kanıt kontrol ediyor: "Evet ruhsat sahibi Mehmet"
  • "Approve" butonuna tıklıyor
✅ Sonuç:
  • Talep status = approved
  • Restaurant'ın sahibi = Mehmet
  • Item'in verification_status = claimed
  • Mehmet artık restoran'ı düzenleyebiliyor

📖 Başarısız Örnek: Sahte Claim

Senaryo

Ahmet bir restoran buldu ve "bu benim" dedi ama kanıt yok.

Ahmet'in Yaptığı:
  • Form dolduruyor
  • Evidence: Boş (hiç kanıt yok)
  • Note: "Evet bu benim"
❌ Admin'in Yaptığı:
  • Kanıt yok → Şüpheli
  • "Reject" tıklıyor
  • Note ekliyor: "Kanıt alınız"
⚠️ Sonuç:
  • Talep status = rejected
  • Restoran'ın sahibi yok
  • Ahmet tekrar deneyebilir (kanıtla)

🔀 Claim Durumları

Durum Anlamı Sonrası
pending Talep yapıldı, admin incelemedi Admin kontrol edebilir
approved Admin onayladı Sahiplik verildi, işlem bitti
rejected Admin reddetti Yeni talep açılabilir
cancelled Talep iptal edildi Yeni talep açılabilir

⚙️ Teknik Detaylar

Tablo: catalog_claim_requests

{ id: UUID, // Talep ID item_id: UUID, // Hangi item? requested_by_user_id: UUID, // Kim talep etti? claim_type: 'ownership', // Sahiplik talebii evidence: JSON, // Kanıt dosyaları note: string, // Açıklama status: 'pending|approved|rejected|cancelled', reviewed_by_user_id: UUID, // Kim inceledi? reviewed_at: timestamp, // Ne zaman incelendi? }

RPC Fonksiyonları

1. submit_catalog_claim_request()

  • Talep yapar
  • User tarafından çağrılır
  • Evidence ve note ekler

2. review_catalog_claim_request()

  • Talebi inceler
  • Admin (moderator) tarafından çağrılır
  • Approve / Reject / Cancel karar verir

🔧 Admin Kontrolü

Admin Neler Yapabilir?

İşlem Ne Yapar? Nerede?
Talep İncele Kanıt ve detayları görebilir /admin/approvals
Approve Sahiplik verilir /admin/approvals → Approve
Reject Talep reddedilir /admin/approvals → Reject
Not Ekle Neden karar verdi not düşebilir /admin/approvals → Review Note

SSS: Admin Soruları

S: Kanıt nasıl kontrol ederim?
C: Evidence field'inde JSON dosyalar var. Fotoğraf URL'leri, belgeler. Admin panelinde preview görebiliyor.
S: Yanlış onay verdiysem?
C: Talep approved/rejected olduktan sonra değiştirilemez. Yanlışsa yeni talep açılması istenir.
S: Claim Rejected ise?
C: Kullanıcı daha iyi kanıt dengan yeni talep açabilir.

✅ Claim Approved Olduktan Sonra?

1. Sahiplik Verilir
Item'in sahip alanı güncellenir.
2. Status Değişir
Item'in verification_status = "claimed"
3. Yönetim Yetkisi
Sahibi item'i düzenleyebiliyor:
  • Açıklamayı güncelleyebilir
  • Fotoğraf ekleyebilir
  • İletişim bilgisini değiştirebilir
  • Yayın durumunu (published/draft) kontrolleyebilir
4. Audit Log
Tüm işlem sistem'de kaydedilir (kim, ne zaman, neler).

CorteQS Fin — Claim Sistemi Rehberi
