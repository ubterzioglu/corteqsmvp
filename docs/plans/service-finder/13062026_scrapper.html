1 Tavily hesabı + API anahtarı 10 dk

  1. app.tavily.com → kayıt ol (Free plan: 1.000 kredi/ay — canary için fazlasıyla yeter).
  2. Dashboard → API Keys → anahtar oluştur, kopyala (tvly-...).
  3. Bu anahtar yalnız Coolify'a girilecek (Adım 3) — repoya / .env.example'a asla yazma.

2 SerpAPI hesabı opsiyonel 10 dk

  1. serpapi.com → kayıt ol (Free: 250 arama/ay).
  2. Account → API Key'i kopyala.
  3. İstemiyorsan atla: Coolify'da SERVICE_FINDER_ENABLE_SERPAPI_FALLBACK=false ayarla ve SERPAPI_API_KEY'i hiç girme. (Admin panelden de Sağlayıcılar → SerpAPI → Etkin'i kapat.)

3 Coolify'da worker uygulamasını kur 15 dk

Mevcut web uygulamasına (App 1) DOKUNMA. Yeni, ikinci bir uygulama:

  1. Coolify → + New → Application → kaynak repo: ubterzioglu/corteqsmvp, branch main.
  2. Base Directory: workers/service-finder
    • Build Pack: Nixpacks (otomatik algılar) · Build: npm ci && npm run build · Start: npm run start
    • Base directory sorun çıkarırsa: Build Pack = Dockerfile, Dockerfile Location = workers/service-finder/Dockerfile, Base Directory = workers/service-finder.
  3. Environment Variables sekmesine şunları gir:
    DeğişkenDeğer
    SUPABASE_URLhttps://injprdrsklkxgnaiixzh.supabase.co
    SUPABASE_SERVICE_ROLE_KEYApp 1'dekiyle aynı (Supabase Dashboard → Settings → API)
    GEMINI_API_KEYMevcut anahtarın (App 1 / edge function'larla aynı)
    TAVILY_API_KEYAdım 1'deki anahtar
    SERPAPI_API_KEYAdım 2'deki anahtar (opsiyonel)
    SERVICE_FINDER_WORKER_IDsf-worker-coolify-1
  4. Health Check'i kapat (worker HTTP servis etmez; açık kalırsa Coolify sürekli restart atar).
  5. Deploy → loglarda şunu görmelisin: Service Finder worker başladı: sf-worker-coolify-1.

4 Canary işi çalıştır 10 dk + ~5 dk bekleme

  1. https://corteqs.net/admin/service-finder → giriş yap (admin hesabınla).
  2. "Hızlı İş Oluştur":
    • Şablon: Doktor · Başlık: Canary - Dortmund doktor
    • Lokasyon: Dortmund, Nordrhein-Westfalen, Germany · Şehir: Dortmund · Ülke: DE
    • Soft cap: 0.10 · Hard cap: 0.20 (canary'de düşük tut!)
  3. "İşi Kuyruğa Al" → iş listesinde Kuyrukta → worker alınca Çalışıyor olur (sayfa 4 sn'de bir kendini yeniler).
  4. İş detayında kontrol et:
    • Maliyetler sekmesi: her arama/ekstraksiyon/sınıflandırma için satır düşüyor mu?
    • Kaynaklar sekmesi: Robots engelledi durumlu kaynak varsa robots kontrolü çalışıyor demektir.
    • Bütçe çubuğu doluyorsa ve $0.20'ye varınca iş Bütçe durdurdu'ya geçiyorsa hard-stop çalışıyor.
    • Normal bitişte durum İncelemede olur ve Adaylar sekmesi dolar.

5 Aday yayınla ve katalogda doğrula 5 dk

  1. İş detayı → Adaylar → bir adaya tıkla → çekmecede kanıt alıntılarını ve alanları kontrol et.
  2. Gerekirse isim/hizmet/dil düzelt → Kataloğa Yayınla.
  3. Toast'ta catalog_item_id görünür; /admin/data'da kaydı ara — kayıt, iletişim, lokasyon ve dilleriyle birlikte gelmiş olmalı.

6 Tip dosyasını yenile 5 dk acil değil

.env.local'deki SUPABASE_ACCESS_TOKEN süresi dolmuş (Unauthorized). Yenisi için:

supabase login          # tarayıcı açılır, onayla
supabase gen types typescript --project-id injprdrsklkxgnaiixzh --schema public > supabase/types.ts
npm run test            # hiçbir şeyin kırılmadığını doğrula
git add supabase/types.ts && git commit -m "chore: regenerate supabase types (service finder)"
Not: --db-url varyantı Docker ister (makinede yok) — yukarıdaki Management API yolu Docker'sız çalışır.
Bu adım acil değil; UI tipsiz supabase.rpc(...) deseniyle çalışıyor (repo geleneği, backlog B1).
